# Story 2 Implementation Plan

## Story Being Implemented

`Story 2: Explicit `density_matrix` Selection Activates The Supported Anchor
Noisy VQE Path`

This is a Layer 4 engineering plan for implementing the second behavioral slice
from `TASK_1_STORIES.md`.

## Scope

This story turns explicit `backend="density_matrix"` into a real supported
execution path for the frozen Phase 2 anchor workflow:

- the supported entry point remains
  `qgd_Variational_Quantum_Eigensolver_Base`,
- the mandatory source circuit remains the default `HEA` circuit produced by
  `Generate_Circuit()`,
- the mandatory target remains density-matrix execution built around
  `NoisyCircuit`, `GateOperation`, and ordered `NoiseOperation` insertions,
- the supported workflow family remains noisy XXZ VQE with local `Z` field,
- and the positive path must be demonstrable at least on the supported 4- and
  6-qubit anchor cases.

Out of scope for this story:

- making `density_matrix` the default backend,
- full `qgd_Circuit` or arbitrary `Gates_block` parity,
- the full unsupported-case and hard-error matrix owned by Story 3,
- the full artifact provenance package owned by Story 4,
- and complete closure of the Task 2 observable-validation package or Task 3
  bridge-support matrix beyond the thin vertical slice required here.

## Dependencies And Assumptions

- Story 1 is already in place: explicit backend selection exists, omitted
  backend remains `state_vector`, and `density_matrix` is a valid explicit mode.
- `NoisyCircuit` already exposes the required operation substrate for this story
  through ordered gate and noise additions, including `add_U3()`,
  `add_CNOT()`, `add_depolarizing()`, `add_amplitude_damping()`, and
  `add_phase_damping()`.
- Story 2 is allowed to implement the thinnest vertical slice of Task 2
  observable behavior and Task 3 bridge behavior needed to make explicit
  density-backend selection real, but it does not fully close those tasks.
- Qiskit Aer remains the external density-matrix reference for positive anchor
  validation cases.

## Engineering Tasks

### Engineering Task 1: Add Density-Backend Runtime Dispatch For Supported Anchor Runs

**Implements story**
- `Story 2: Explicit density_matrix Selection Activates The Supported Anchor Noisy VQE Path`

**Change type**
- code

**Definition of done**
- The VQE core can branch into a density-backed execution path when the
  effective backend mode is `density_matrix`.
- Explicit `density_matrix` selection no longer ends at API normalization only;
  it reaches a distinct runtime branch for supported anchor workflows.
- The legacy `state_vector` path remains intact for omitted backend and explicit
  `state_vector`.

**Execution checklist**
- [ ] Add backend dispatch in the VQE core entry points used by fixed-parameter
      energy evaluation and the optimization loop.
- [ ] Isolate the density branch behind dedicated helper functions rather than
      scattering conditionals across existing state-vector logic.
- [ ] Keep unsupported density requests clearly separable so Story 3 can add the
      final hard-error behavior without rewriting the Story 2 path.

**Evidence produced**
- Focused positive test proving explicit `density_matrix` enters a distinct
  density-backed branch.
- Reviewable code path rooted in
  `Variational_Quantum_Eigensolver_Base`.

**Risks / rollback**
- Risk: branch placement too high in the VQE stack can regress legacy
  state-vector behavior.
- Rollback/mitigation: keep density dispatch narrowly scoped and default all
  existing callers to the current state-vector implementation.

## Exit Criteria

Story 2 is complete only when all of the following are true:

- explicit `density_matrix` selection reaches a distinct density-backed runtime
  path through `qgd_Variational_Quantum_Eigensolver_Base`,
- the default `HEA` anchor circuit can be lowered into a supported
  `NoisyCircuit` path with required ordered local noise insertion,
- supported 4- and 6-qubit anchor cases execute through the density path,
- at least one reproducible 4- or 6-qubit optimization trace demonstrates
  end-to-end density-backed execution,
- and validation evidence can distinguish the density path from the legacy
  state-vector path.

## Implementation Notes

- `NoisyCircuit` already exposes the required gate and local-noise API surface
  for the Story 2 vertical slice, so implementation should reuse that substrate
  rather than introduce a second density-circuit abstraction.
- The current density-module examples and tests already show gate-plus-noise
  execution behavior that can be reused as reference fixtures for Story 2.
- The explicit ordered-noise concept should stay aligned with the manual
  insertion semantics used in the Qiskit Aer comparison harness.
- Story 2 should stay intentionally narrow: default `HEA`, the anchor XXZ
  Hamiltonian family, and the required local noise models only.
- Full unsupported-case behavior and full backend provenance packaging remain
  later-story work even though Story 2 should produce enough evidence to prove
  the density path is genuinely exercised.
